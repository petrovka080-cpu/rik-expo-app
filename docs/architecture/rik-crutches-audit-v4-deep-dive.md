# RIK Platform: Экстремально глубокий Read-Only аудит (Срез по коду v4)

Этот отчет базируется на глубоком прочесывании кодовой базы с помощью `grep` поиска. Несмотря на успехи в базовой стабилизации проекта, фактический срез кода обнажает огромный пласт оставшихся платформенных костылей, которые будут блокировать масштабирование.

## 1. Executive Summary (Сухие цифры)

Проект компилируется "в зеленую" (`tsc --noEmit`), однако под капотом бизнес-логика удерживается за счет подавления типов в рантайме и хардкод-таймеров.

* **`as any` / `as unknown as`**: Более 300 вхождений в слое API и парсинга (в 4 раза больше, чем должно быть в здоровом проекте).
* **`setTimeout` (Race Condition Hacks)**: Обнаружено более 45 мест ручной синхронизации UI.
* **`supabase.from` / `supabase.rpc`**: Вызываются напрямую из `src/screens/...` более 100 раз.
* **Общий вердикт:** Фаза "слепоты типов" пройдена, но проект находится на стадии "архитектурного монолита". Слой работы с БД жестко спаян с визуальными компонентами React (UI Coupling).

## 2. Карта системных костылей (Основано на коде)

### 2.1. UI ↔ DB Coupling (Сцепка интерфейса и данных) - P0
* **Где найдено:** 
  * `src/screens/foreman/ForemanSubcontractTab.tsx` (прямой вызов `supabase.from("requests" as any)`)
  * `src/screens/director/director.data.ts` (`supabase.rpc("list_director_items_stable")`)
  * `src/screens/warehouse/warehouse.seed.ts` (Upsert прямо из UI утилит Склада).
* **Риск:** UI-компоненты принимают решения об именах таблиц и процедур. Любое изменение в бэкенд-схеме приведет к ручному поиску и переписыванию `.tsx` файлов.
* **Масштаб:** 100+ вызовов прямо в `src/screens`. Полное отсутствие единого Service Layer.

### 2.2. Type Hacks (Скрытый Schema/ID Drift) - P1
* **Где найдено:**
  * `src/screens/buyer/hooks/useBuyerSuppliers.ts` (Циклы вида `for (const row of contractors as any[])`)
  * `src/screens/director/director.finance.panel.ts` (`(spendFiltered as any[]).filter(...)`)
  * `src/screens/foreman/ForemanSubcontractTab.tsx` (Сотни ручных парсингов: `(rq.data as any).request_no`)
* **Риск:** Компилятор TypeScript показывает "0 ошибок", но в реальности `any` полностью отключает проверку полей. Проект упадет в Runtime, если API вернет `null` вместо объекта `rq.data`.
* **Масштаб:** 333 необработанных вхождения по всему проекту.

### 2.3. Runtime-костыли (Таймеры и Госты UI) - P2
* **Где найдено (45+ мест):**
  * `src/components/map/CatalogSearchModal.tsx` (`setTimeout(() => void runSearch(), 250)`)
  * `src/screens/accountant/components/ActivePaymentForm.tsx` (задержки фокуса дропдаунов `NEXT_TICK_MS`)
  * `src/lib/pdfRunner.ts` (задержки в 500мс перед очисткой ФС `setTimeout(cleanup, 500)`)
* **Риск:** Приводит к непредсказуемым гонкам компонентов на слабых устройствах. На быстрых смартфонах модалка закроется быстрее таймера, вызвав "Обновление размонтированного компонента".

### 2.4. PDF Engine (Монолитность и Строки) - P1
* **Где найдено:** `src/lib/pdfRunner.ts` (ручная сборка HTML)
* **Риск:** Гигантские HTML-строки актов "убивают" память Hermes GPU/CPU на мобильных телефонах. При объемных накладных приложение будет крашиться от нехватки памяти (OOM - Out of Memory).

### 2.5. Архитектурное дублирование (Ролевой копипаст) - P1
* **Где найдено:** Семейство `*SubcontractTab.tsx` и `director.proposal.*.ts`.
* **Риск:** Копирование одних и тех же RPC-вызовов (`supabase.rpc("subcontract_create_draft")`) в логике разных ролей. Утроенное время на поддержку бизнес-флоу.

## 3. Hotspots (Красные зоны кодовой базы)

1. **`app/(tabs)/warehouse.tsx` (>1500 строк)**
   * Слияние: UI-Рендер + `supabase.from` + `PDF Сигнатуры`. Трогать этот файл в рамках фичи крайне опасно.
2. **Сеть хуков `useBuyerSuppliers.ts` / `useBuyerSelectionActions.ts`**
   * Содержит массивные циклы с `as any`. Выполняет N+1 запросов к БД без кэширования, что замедляет работу Снабженца.
3. **`ForemanSubcontractTab.tsx` (800+ строк)**
   * Пятно `as any` кастов. Прямая перезапись полей (`.update(directPatch as any)`) без валидации DTO.

## 4. Что уже стабильно (Не трогать, работает хорошо!)

1. **Типизация Базы:** Файл `database.types.ts` существует и ядро Supabase-клиента его понимает. (Проблема только в том, что UI обходит его через `as any`).
2. **Expo Router:** Роутинг работает четко, без наслоения стейт-стеков.
3. **Изоляция Директора:** Директор живет в своем монолитном табе "Контроль", надежно защищен от перекрестных бизнес-связей с прорабами и снабженцами. **Это жесткое правило, дробить роль Директора на под-экраны категорически запрещено!**

## 5. Классификация и Точный Roadmap

### Phase 0: Safe cleanup (Критично, без смены логики)
* Вычистить остатки "кракозябр" из `useBuyerSuppliers.ts` (убрать fallback `Alert.alert` на невнятные объекты).
* Обернуть все `console.log(payload)` в карточках контракторов в `if (__DEV__)`, чтобы не убивать FPS скролла в проде.

### Phase 1: Local adapter fix (Типизация хуков)
* Заменить 333 вхождения `as any` в `src/screens/...` на использование `database.types.ts`.
* Все `supabase.from()` вызовы, находящиеся прямо внутри `useEffect` визуальных компонентов (например в Складе), обернуть в локальные хуки (без переноса в другие файлы).

### Phase 2: Structural refactor later (Сервисный слой)
* Внедрение `src/services/` или глобального `React Query` стора. Полная отвязка React UI от знаний о таблицах PostgreSQL. Экран должен запрашивать: `useInventory()`, а не `supabase.from('v_warehouse_stock')`.
* Слияние ролевых `*SubcontractTab.tsx` в единый визуальный модуль.

### Keep temporarily (Категорически не трогать!)
* **Генерация PDF (`pdfRunner.ts`):** Оставить ручную склейку HTML до выделения бюджета на переписывание модуля на React Components SSR. Изменение `setTimeout` хаков в `pdfRunner.ts` сейчас приведет к черным экранам при шеринге.
* **Монолит Директора:** Оставить 100% логики внутри `DirectorDashboard.tsx` и его вложенных компонентов.

---
**Итог глубокого среза:** Проект жизнеспособен и компилируется, но архитектурно похож на "карточный домик", склеенный `setTimeout` таймерами и `as any`. Для масштабирования ролей критически необходима **Фаза 1 (Отвязка БД от UI)**.
