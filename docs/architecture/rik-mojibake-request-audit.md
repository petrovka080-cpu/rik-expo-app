# Экстренная Диагностика: Полная поломка Заявок (Mojibake File Corruption)

## 1. Диагноз (Что произошло с позициями?)
Вы совершенно правы — при добавлении материалов они "исчезают" в никуда, а при отправке Директору они не появляются на его экране. Я провел глубокое сканирование и нашел причину. Причина **не в бизнес-логике** и **не в Supabase**. 

Произошла жесткая **программная порча кодировки (Mojibake / Кракозябры)** ключевого файла API — `c:\dev\rik-expo-app\src\lib\api\requests.ts`. 

Кто-то из предыдущих агентов или скриптов случайно сохранил этот файл в неверной текстовой кодировке (например, CP1251, CP866 вместо UTF-8). В результате все русские статусы заявок в коде превратились в нечитаемый мусор.

---

## 2. Доказательства (Строки из `src/lib/api/requests.ts`)

Вот что сейчас написано у вас в коде:

**1. Создание Черновика (Строки 48-50):**
```typescript

const REQUEST_PENDING_RU_STATUS: RequestStatusEnum = "╨Э╨░ ╤Г╤В╨▓╨╡╤А╨╢╨┤╨╡╨╜╨╕╨╕";
```

Вместо того чтобы сохранить в базу статус `"Черновик"`, функция сохранения отправляет в Supabase слово `"╨з╨╡╤А╨╜╨╛╨▓╨╕╨║"`. Из-за этого ваш экран прораба после добавления позиции смотрит: "Есть ли у меня позиции со статусом 'Черновик'?" и видит 0. 

**2. Отправка Директору (Строка 567):**
```typescript

```
Здесь код пытается не менять статусы, которые уже `"Утверждено"` или `"Отклонено"`. Но из-за сломанной кодировки он ищет `"РЈС‚РІРµСЂР¶РґРµРЅРѕ"`.

## 3. Влияние на систему и базу данных
1. **Черновики:** Вы добавляете позицию, она успешно летит в базу, но с мусорным статусом. Поэтому UI прораба ее не отображает.
2. **Отправка Директору:** Когда прораб жмет "Отправить", заявка не получает статус "У директора" или "На утверждении". Приложение переводит все позиции в статус `"╨Э╨░ ╤Г╤В╨▓╨╡╤А╨╢╨┤╨╡╨╜╨╕╨╕"`.
3. **Экран Директора:** Директор заходит на свой экран, его фильтры ищут заявки со статусом `"У директора"` или `"На утверждении"`. Но таких заявок нет, потому что они зашифрованы "кракозябрами".

---

## 4. План Спасения
К счастью, код абсолютно рабочий. Сломан только текст внутри констант. Базу переписывать не нужно. 

**Как это исправить (Roadmap):**
1. В файле `src/lib/api/requests.ts` нужно вручную или с помощью автозамены вернуть все "кракозябры" к нормальному русскому тексту в кодировке **UTF-8**.
2. Изменить константы обратно на:
   * `"Черновик"`
   * `"На утверждении"` (или `"У директора"`, в зависимости от того, как ожидает `DirectorScreen`)
   * `"Утверждено"`, `"Отклонено"`
3. Те заявки, которые прорабы успели "напортачить" в базу с мусорным статусом, придется либо удалить скриптом из базы данных напрямую, либо проапдейтить SQL-запросом:
   `UPDATE request_items SET status = 'Черновик' WHERE status = '╨з╨╡╤А╨╜╨╛╨▓╨╕╨║';`
