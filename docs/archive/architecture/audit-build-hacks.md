# Audit: Build Hacks & Patch Scripts

**Criticallity: P0 (CRITICAL)**

Самая опасная находка в проекте — наличие целого слоя "Mutation Scripts", которые модифицируют исходный код проекта с помощью "регулярок" (`regex`) на лету.

## 1. Категория: TS Patch Scripts (Auto-Ignore)

**Найденные скрипты в корне проекта:**
- `fix-ts-ignore.js`
- `fix-ts-hard.js`, `fix-ts.js`, `fix-ts2.js`, `fix-ts3.js`
- `auto-ignore.js`
- `fix-accountant-cast.js`
- `fix-syntax.js`

**Что происходит (Описание проблемы):**
При несовпадении типов, вместо того чтобы исправлять интерфейсы и DTO в коде, разработчики (или агенты) написали NodeJS-скрипты, которые читают файлы перед компиляцией, находят красные строки и автоматически вставляют над ними `// @ts-ignore` (либо меняют типы автозаменой). 
Это абсолютно ломает всю экосистему и доверие к коду. Версия файлов в IDE и версия файлов на этапе сборки EAS Build — это два разных проекта. 

## 2. Категория: UI/Logic Regex Patching

**Найденные скрипты:**
- `fix-stars.js`
- `fix-rik-api.js`
- `fix-incoming-items.js`
- `fix-foreman-props.js`
- `scripts/fix-web-dom.js`

**Что происходит:**
Скрипты агрессивно переписывают логику компонентов, импорты и манипуляции с DOM/WebView для адаптации под разные платформы. Это крайняя форма "костыля".

**Roadmap устранения без смены логики:**
1. Очистить `package.json` (`postinstall` и `prebuild` хуки) от запуска этих скриптов.
2. Переместить логику ветвления платформ из RegEx-мутаций в стандартные механизмы React Native (`Platform.OS === 'web'` или использование `.web.ts` / `.native.ts` extensions).
3. Удалить эти файлы-мутаторы из репозитория. Зафиксировать типы в коде.
