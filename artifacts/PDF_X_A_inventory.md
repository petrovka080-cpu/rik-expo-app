## Inventory Conclusions

- Heavy compute is not concentrated in `pdf-viewer`. The viewer receives descriptors/URIs after upstream source and render work.
- The highest-risk paths are server-rendered report PDFs that run report source + HTML + Puppeteer + upload on each cold click with no durable versioned artifact contract.
- Some source RPCs already expose `version` or `generated_at`, but that metadata is not yet a full freshness/invalidation contract for cached PDF artifacts.
- Director finance management has partial mitigation through source/render prefetch and short-lived in-memory caches. It still lacks durable freshness, but it is less severe than paths with no reuse at all.
- Warehouse report PDFs are isolated behind `warehouse-pdf`, but the all-period material/object-work family still looks like a precompute candidate because it combines DB aggregation and render on click.
